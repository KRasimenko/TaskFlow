# Спецификация прецедентов

## UC-01 «Войти в систему»

| Поле | Значение |
|------|----------|
| Актёр | Пользователь |
| Предусловие | Приложение установлено, есть аккаунт |
| Основной поток | 1. Пользователь вводит email и пароль. 2. Клиент отправляет POST /api/auth/login. 3. Сервер проверяет учётные данные. 4. Сервер возвращает JWT. 5. Клиент сохраняет токен и открывает главный экран. |
| Альтернатива | Неверный пароль → сообщение об ошибке |
| Постусловие | Пользователь авторизован |

## UC-06 «Создать задачу»

| Поле | Значение |
|------|----------|
| Актёр | Пользователь |
| Предусловие | Пользователь авторизован |
| Основной поток | 1. Нажатие FAB «+». 2. Заполнение формы. 3. POST /api/tasks с JWT. 4. Сервер сохраняет в БД. 5. Обновление списка на главном экране. |
| Альтернатива | Нет сети → сохранение в локальный кэш с последующей синхронизацией |
| Постусловие | Задача создана |

## UC-07 «Удалить задачу»

| Поле | Значение |
|------|----------|
| Актёр | Пользователь |
| Предусловие | Задача существует |
| Основной поток | 1. Открыть карточку задачи. 2. Нажать «Удалить». 3. Подтвердить. 4. DELETE /api/tasks/{id}. 5. Возврат на главный экран. |
| Постусловие | Задача удалена из БД |
